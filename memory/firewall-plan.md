# План сетевой безопасности (08.05.2026)

## Текущее состояние
- **Сервер:** Ubuntu 24.04 LTS, AMD, RTX 3090, 32GB RAM
- **Сеть:** 1 порт (enp3s0, 192.168.0.181)
- **Роутер:** D-Link DIR-842 (192.168.0.1)
- **Фаервол:** ❌ нет
- **DNS:** systemd-resolved

## Схема
```
Интернет → [Lora] → D-Link DIR-842 → WiFi → устройства
```

## 🔥 Выбор фаервола

### Вариант 1: nftables (рекомендую)
- Нативный для Ubuntu 24.04
- Заменяет iptables
- Быстрый, лёгкий
- Можно настраивать через Lora

### Вариант 2: pfSense
- Отдельный фаервол
- Требует отдельный сервер/VM
- Сложнее в настройке

### Вариант 3: OPNsense
- Как pfSense, но новее
- Требует отдельный сервер

**Рекомендация:** nftables — нативный, лёгкий, без лишнего

## 📋 Что нужно купить

### Обязательно:
1. **USB Ethernet адаптер** (Realtek RTL8153) — ~800₽
   - Второй сетевой порт для сервера
   - Один порт — WAN (интернет)
   - Второй порт — LAN (в роутер)

### Опционально:
2. **USB WiFi адаптер** — ~500₽
   - Для мониторинга WiFi
   - Не обязательно

## ⚙️ Настройка

### Фаза 1: Подготовка (покупка адаптера)
- Купить USB Ethernet адаптер
- Подключить к серверу
- Проверить, определился ли второй порт

### Фаза 2: Конфигурация сети
- Порт 1 (enp3s0): WAN → интернет
- Порт 2 (USB адаптер): LAN → роутер
- NAT/маршрутизация через nftables

### Фаза 3: Фаервол правила
- Входящий трафик: только разрешённые порты
- Исходящий: разрешить всё из LAN
- Блокировка нежелательных портов
- Логи

### Фаза 4: Мониторинг
- Lora будет отслеживать трафик
- Уведомления в Telegram при аномалиях
